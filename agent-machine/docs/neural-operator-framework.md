# Neural Operator Framework — integration architecture

Neural operators (FNO, DeepONet) learn the *solution operator* of a PDE — they return a solution field in one
forward pass, resolution-free. They fill exactly one slot in our stack: **the executable for the canon's PDE
entries** (graduate physics/EE — heat, wave, Schrödinger, Poisson, diffusion, fields) where sympy has no closed
form. This framework makes them pluggable, **verified**, and evidence-emitting, so they register as tools the
same way sympy does. The numerical v0 (`scripts/neural_operators.py`) works *today*; FNOs drop in when trained.

> Assumption: "Prometheus" = the reasoning/orchestration layer that routes a problem to tools, runs them, and
> emits reasoning-evidence. The framework is agnostic to its exact shape — adjust the seam names if different.

## The contract (one interface, always verified)
```
Operator.solve(PDESpec) -> field          # numerical | fno | deeponet — same signature
pde_residual(spec, field) -> float        # plug the solution back into the PDE; 0 = exact
solve_pde(spec) -> PDESolution{ field, backend, residual, verified, confidence, receipt }
```
The **residual gate** is the load-bearing idea: a learned operator's output is never trusted blindly — it's
verified against the PDE it claims to solve, and falls back to numerical (or abstains) if the residual exceeds
tol. This is the verified-compute discipline extended to neural surrogates: a wrong FNO field cannot pass.

## Integration seams
```
        problem (physics/EE)
              │
   ┌──────────▼───────────┐
   │  CSKG routing         │  concept → PDE family   (e.g. "heat transfer"/"conduction" → heat;
   │  (lib/cskg.ts)        │   "vibrating string"/"EM wave" → wave; "bound state" → schrodinger)
   └──────────┬───────────┘
   ┌──────────▼───────────┐
   │  CANON tool registry  │  the PDE canon entry (Schrödinger, heat, Maxwell) names its operator FAMILY;
   │  (canon + symbol-map) │   the entry's "executable" IS the operator (vs sympy for closed-form entries)
   └──────────┬───────────┘
   ┌──────────▼───────────┐
   │  BRAIN (RAG)          │  retrieves the problem SETUP — parameters, boundary/initial conditions, domain —
   │                       │   and assembles the PDESpec
   └──────────┬───────────┘
   ┌──────────▼───────────┐
   │  neural_operators     │  solve_pde(spec): route family → (FNO if trained, else numerical) → VERIFY
   │                       │   residual → confidence → answer
   └──────────┬───────────┘
   ┌──────────▼───────────┐
   │  PROMETHEUS           │  registers solve_pde as a governed TOOL; on a PDE-class problem it dispatches,
   │  (orchestration)      │   takes the verified field, and emits the Receipt into the reasoning-evidence
   │                       │   fabric (replayable, attestable) — same as every other arm/tool
   └──────────────────────┘
```

- **Knowledge graph**: each PDE equation is an entity; its operator is the equation's `executes-with` edge; a
  solved problem links problem → equation → operator → receipt. The operator becomes a first-class capability
  node, not a hidden function.
- **Compute arm / verified-compute**: the dispatcher (the Gödel-structure idea) routes algebraic → sympy,
  PDE-class → `neural_operators`. One registry, two backends, both verified.
- **Evidence**: every `solve_pde` emits `{tool, family, backend, residual, verified, confidence}` — so a neural
  surrogate's answer carries its own proof-of-trust into the fabric.

## Pluggability + training path
- **v0 (now)**: finite-difference numerical operators for heat/diffusion/poisson (+ wave/Schrödinger schemes).
  The framework is live and verified end-to-end on these.
- **v1 (Phase-N ML project)**: train an FNO/DeepONet per family on data generated by the numerical solver
  (input: BCs/ICs/params → output: field), validate by held-out residual, register it — callers unchanged.
  Wins: one-pass, resolution-free, differentiable (useful for the Academy's interactive simulation + inverse
  problems).

## Honest scope
Narrow but real. This serves (a) **verified-compute on genuinely-PDE grad-physics problems** and (b) the
**Alexandrian Academy's interactive physics simulation**. It does **not** touch the knowledge-graph/RAG/glossary
work, and it moves only the small PDE-solving slice of the board — so it's a Phase-N tool, built and ready,
**not** ahead of the board loop. The framework + numerical v0 are done; the FNOs are the upgrade.
