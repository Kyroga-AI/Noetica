# Noetica Onboarding Remediation Surface

Status: Phase 1H desktop hardening tranche

## Purpose

The desktop shell should explain missing, disabled, deferred, or error states with concrete next actions.

## Implementation

Guidance is generated by:

```text
lib/client/remediation.ts
```

and rendered in:

```text
components/status/RuntimeStatus.tsx
```

The current surface remains read-only. It adds operator guidance to the existing top-bar status card.

## Current examples

The status card may show guidance for:

- fallback runtime boundary
- provider setup
- SourceOS route readiness
- Agent Machine readiness
- Prophet Mesh readiness

## Non-goals

- No daemon implementation.
- No provider migration.
- No SourceOS route integration.
- No Agent Machine handshake.
- No packaging implementation.
- No Electron fallback.

## Next tranche

Begin production packaging metadata: icon requirements, bundle identity, signing and notarization plan, Linux desktop entry plan, and provenance/SBOM plan.
